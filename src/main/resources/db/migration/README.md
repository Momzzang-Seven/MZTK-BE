# DB Migration Guide

이 디렉토리는 **데이터베이스 스키마 변경을 코드처럼 버전 관리**하기 위한 공간입니다.  
초기 개발 단계에서는 비어 있을 수 있으며, **첫 배포 이후부터는 필수적으로 사용**됩니다.

---

## 목적

- DB 스키마 변경 이력을 명확하게 관리한다
- 개발/CI/운영 환경의 DB 상태를 동일하게 유지한다
- JPA의 `ddl-auto`에 의존한 **암묵적 변경을 방지**한다
- 운영 DB 변경 시 발생할 수 있는 **사고를 예방**한다

---

## 적용 도구

- **Flyway** (권장)
- 또는 Liquibase

본 프로젝트는 Flyway 사용을 기준으로 설계되어 있습니다.
